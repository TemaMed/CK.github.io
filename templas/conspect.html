Краткий конспект

Что на самом деле происходит, когда вы вводите адрес сайта и нажимаете Enter?
Сайт обслуживается программой, которая называется веб-сервер. Не просто сервер, а веб-сервер, и по умолчанию, или традиционно, у веб-сервера порт - 80.
Веб-серверы тоже могут быть похожи на коммуналки. В этом случае веб-сервер обслуживает сразу несколько сайтов, и такие сайты называются виртуальными хостами или virtual хостами.
HTTP-пакеты бывают двух видов: это HTTP-запрос и HTTP-ответ. Они отличаются набором заголовков.HTTP-запрос – это тот пакет, который отправляет браузер серверу, чтобы сервер дал
Первый и, наверное, самый главный заголовок — это хост, в котором пишется то самое имя сайта, которое нужно серверу нам отдать.
То есть песочница — это, по сути, удобный инструмент общения для веб-разработчиков, позволяющий обмениваться кусочками кода и искать ошибки друг у друга.
Язык HTML
HTML — это не язык программирования, а язык разметки, т. е. он описывает содержание веб-страницы, но не описывает его поведение, и состоит он из так называемых HTML-тегов. HTML-тег пишется следующим образом: сначала идет знак меньше, затем следует имя тега и затем следует знак больше.
CSS (/siːɛsɛs/ англ. Cascading Style Sheets — каскадные таблицы стилей) — формальный язык описания внешнего вида документа, написанного с использованием языка разметки.
Типовая структура проекта по вёрстке выглядит так:
● обычно все файлы проекта держат в какой-то папке с именем проекта.
● мы назвали эту папку просто project.
● в корне папки лежат файлы с расширением .html.
● главную страницу обычно помещают в файл с названием index.html.
Подведём промежуточный итог:
● чтобы выбрать товар, вам нужно фильтровать и сортировать товары в каталоге
● для этого вам нужно передавать на сервер параметры для фильтрации и сортировки
● такие параметры чаще всего передаются в адресной строке браузера (чтобы ссылкой можно было легко поделиться или просто сохранить в закладки).
● в HTML-разметке это реализуется либо с помощью ссылок, либо с помощью форм с методом GET.
● когда вы щёлкаете на ссылку, в адрес которой зашиты параметры, либо отправляете форму, браузер переходит по новому адресу, а для этого отправляет на сервер HTTP-запрос
● сервер получает HTTP-запрос, в котором содержатся эти параметры и передаёт его в серверную программу-обработчик (например, написанную на языке PHP)
● эта программа в соответствии со значениями параметров формирует код страницы, который отправляется в ваш браузер.
● И вы видите нужную страницу.
● Конечно, вы можете изменить параметры (щёлкнув по другой ссылке или изменив значения полей формы) и тогда всё начинается по новой.
